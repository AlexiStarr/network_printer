# 打印机控制系统 - 快速启动与测试指南

## 🚀 快速启动

### 前置要求
- Go 1.16+ 
- GCC/Clang（编译C驱动）
- SQLite3
- 现代浏览器（Chrome、Firefox、Safari等）

### 启动步骤

#### 1. 编译C驱动
```bash
cd driver/
gcc -o printer_driver printer_simulator.c driver_server.c main_driver.c -lpthread
```

#### 2. 启动驱动程序
```bash
./printer_driver
# 输出：
# ================================
#   网络打印机驱动程序 v1.0
# ================================
# 驱动程序启动成功，监听端口 9999
```

#### 3. 启动后端服务
```bash
cd backend/
go run main.go
# 或先编译再运行
# go build -o printer_backend
# ./printer_backend
```

后端默认监听 `http://localhost:8080`

#### 4. 打开前端页面
方式A：使用原始HTML（较旧）
```bash
open printer_control.html
```

方式B：使用改进的新前端（推荐）
```bash
open printer_control_improved.html
```

或在浏览器中打开：`file:///path/to/printer_control_improved.html`

## 🔧 常见问题排查

### 问题1：无法连接到驱动程序
```
错误：无法连接到驱动: connection refused

解决：
1. 确保驱动程序已启动
2. 检查驱动监听的端口（默认9999）
3. 检查防火墙设置
4. 查看驱动程序日志
```

### 问题2：WebSocket连接失败
```
错误：WebSocket connection failed

解决：
1. 确保后端服务正在运行
2. 检查 WS_URL 配置（ws://localhost:8080/ws）
3. 浏览器控制台查看详细错误
4. 清除浏览器缓存和Cookie
```

### 问题3：登录失败
```
错误：用户名或密码错误

解决：
1. 确保数据库已初始化
2. 检查默认用户是否存在
3. 确认密码是否正确
4. 查看数据库文件权限
```

### 问题4：操作响应缓慢
```
现象：API响应 > 5秒

解决：
1. 检查系统资源使用（CPU、内存）
2. 查看是否有大量并发请求
3. 检查数据库查询效率
4. 查看网络延迟
```
